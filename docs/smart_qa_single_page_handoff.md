# 智慧问答单页面（数字人 + GPT）UI 交付说明

## 1. 页面定位
- 页面名称：`智慧问答（数字人咨询助手）`
- 页面目标：数字人信任建立、连续问答、结构化沉淀、任务化落地
- 设计分区：`左26% / 中46% / 右28%`

## 2. 布局骨架
- 顶部头部（`qa-header`）：
  - 左：品牌与副标
  - 中：模式切换（问诊/知识问答/方药解析/科研检索）
  - 右：历史会话、通知、模型状态
- 主内容区（`qa-layout`）：
  - 左：数字人舞台、状态卡、实时转写、快捷能力
  - 中：会话头、推荐问题、消息流、过程卡、输入区
  - 右：结果/依据/图谱/任务四Tab + 播报脚本 + 材料清单

## 3. 左侧数字人区
- Avatar Stage：沉浸背景（经络网格 + 粒子流动）
- 状态机：待机 / 倾听 / 思考 / 回答 / 风险提示
- 转写联动：语音识别文本 + 识别标签 + 待确认缺失项
- 快捷能力：语音问诊、上传舌照、上传病历、切换专家数字人

## 4. 中间 GPT 会话区
- 会话头：标题 + 标签 + 收藏/导出/新会话
- 推荐问题 Chips：可点击回填输入框
- 三层消息体系：
  - 用户消息
  - 数字人回复（含操作按钮）
  - 系统过程卡（输入解析、检索命中、规则评分、边界校验）
- 输入区：
  - 工具按钮（图片/文件/麦克风）
  - 多行输入框
  - 场景选择、模式选择
  - 播报开关 + 发送

## 5. 右侧结果沉淀区
- 四个 Tab：`结果 | 依据 | 图谱 | 任务`
- 结果 Tab 卡片顺序：
  1. 主诉摘要
  2. 已识别症状
  3. 舌象/脉象
  4. 证候候选排序
  5. 治法建议
  6. 方药草案
  7. 风险提示
  8. 当前缺失项
- 依据 Tab：古籍/指南/医案证据条目
- 图谱 Tab：症状->证候->治法->方药因果链
- 任务 Tab：病历摘要、辨证报告、专家复核、随访导出等执行入口

## 6. 关键交互状态
1. 欢迎态：推荐问题可见，数字人待机
2. 倾听态：转写实时滚动
3. 推理态：过程卡持续更新
4. 回答态：数字人语音播报 + 右侧结果同步刷新
5. 落地态：任务面板可执行

## 7. 实现映射
- 模板：`app/web/templates/smart_qa.html`
- 脚本：`app/web/static/js/smart_qa.js`
- 样式：`app/web/static/css/styles.css`（Smart QA v2 区块）
- 接口：
  - `GET /smart-qa/scenarios`
  - `POST /smart-qa/ask`
  - `POST /smart-qa/task-execute`
